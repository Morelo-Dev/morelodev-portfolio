Hay una diferencia entre saber programar y saber programar bien. Estos son los recursos que me ayudan a estar en el segundo grupo.

## NeetCode — el mapache que te enseña algoritmos

Si solo vas a abrir una página hoy, que sea esta. [NeetCode](https://neetcode.io) es LeetCode pero organizado por rutas, con videos explicativos para cada problema y una hoja de ruta clara. El mascota es un mapache. El contenido es serio.

Lo que lo hace diferente de LeetCode puro:

- Cada problema tiene un video donde el autor explica la solución paso a paso
- Las rutas están ordenadas por dificultad y tema (arrays, trees, DP, etc.)
- Hay una hoja de ruta "NeetCode 150" con los problemas más frecuentes en entrevistas reales

<InfoBox type="info">
  Si vas a preparar entrevistas técnicas, empieza por la ruta NeetCode 150 en orden. No saltes a problemas difíciles antes de dominar los fáciles.
</InfoBox>

## El resto del stack de recursos

### Para algoritmos

- **[NeetCode](https://neetcode.io)** — el mapache, la ruta, los videos. Ya lo dije.
- **[LeetCode](https://leetcode.com)** — cuando quieres el banco de problemas completo.
- **[Codewars](https://www.codewars.com)** — más divertido, los retos se llaman "kata". Buen ambiente para calentar.

### Para frontend

- **[Frontend Mentor](https://www.frontendmentor.io)** — diseños reales para implementar. Perfecto para portafolio.
- **[CSS Battle](https://cssbattle.dev)** — retos de CSS puros. Adictivo.
- **[JavaScript.info](https://javascript.info)** — la mejor documentación práctica de JS que existe.

### Para arquitectura

- **[Refactoring Guru](https://refactoring.guru/design-patterns)** — patrones de diseño con diagramas. El recurso que debí haber encontrado antes.
- **[roadmap.sh](https://roadmap.sh)** — mapas de ruta para cualquier especialidad. Útil para saber qué no sé.

## Mi rutina real

No practico todos los días, pero tengo un ritmo:

1. **Cuando quiero calentar** — 1 kata en Codewars
2. **Cuando quiero sumar al portafolio** — Frontend Mentor
3. **Cuando quiero entender algo profundo** — Refactoring Guru + NeetCode
