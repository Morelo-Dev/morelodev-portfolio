## Páginas para practicar

Estas son las que abro cuando quiero mantener el músculo activo:

### Para algoritmos y estructuras de datos

- **[LeetCode](https://leetcode.com)** — el estándar. Filtro por "Easy" cuando estoy caliente y "Medium" cuando quiero sufrir un poco.
- **[NeetCode](https://neetcode.io)** — LeetCode pero con rutas organizadas y videos explicativos. Mucho mejor para empezar.
- **[Codewars](https://www.codewars.com)** — más divertido que LeetCode, los retos se llaman "kata". Buen ambiente.

### Para frontend específico

- **[Frontend Mentor](https://www.frontendmentor.io)** — diseños reales para implementar. Perfecto para portafolio.
- **[CSS Battle](https://cssbattle.dev)** — retos de CSS puros. Adictivo y te hace entender el modelo de caja de verdad.
- **[JavaScript.info](https://javascript.info)** — la mejor documentación práctica de JS que existe. Gratis y en varios idiomas.

### Para arquitectura y buenas prácticas

- **[Refactoring Guru](https://refactoring.guru/design-patterns)** — patrones de diseño explicados con diagramas. El recurso que debí haber encontrado antes.
- **[roadmap.sh](https://roadmap.sh)** — mapas de ruta para cualquier especialidad. Útil para saber qué no sé.

<InfoBox type="info">
  No intentes usar todos a la vez. Elige uno para algoritmos y uno para frontend. Consistencia > variedad.
</InfoBox>

## Mi rutina actual

No soy de practicar todos los días, pero sí tengo una rutina:

1. **Lunes** — 1 kata en Codewars para calentar
2. **Miércoles** — Frontend Mentor si tengo algo que agregar al portafolio
3. **Cuando me aburro** — Refactoring Guru para leer sobre patrones
